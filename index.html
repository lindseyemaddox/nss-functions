Functions

NSS repo reference

Functions are reusable code. A pure function is one that is consistent and has no effect on anything else in your system. Impure functions produce different results upon execution or affect other parts of the system when executed.

Functions include:

  • the keyword function
  • a name (you make up) for the function
  • parenthesis that identify it as a function
  • opening and closing curly brackets
  • executable code between the brackets

When data is placed between the parentheses, it is passed to the function that it uses when it executes. This data is called an argument. You can pass any number of arguments, separated by commas, to a function. A variable inside the parentheses in a function statement is known as a parameter. A function can only return a single value through the return statement.

Variable scope is the difference between local and global variables. A local variable is declared inside a function, as a parameter of the function or a variable declared explicitly in the function using a keyword. A global variable is declared in the main body. It's always best to pass values explicitly to functions through arguments. Global variables have no place in functions.

Practice

ChickenMonkey

Write a program that prints the numbers from 1 to 100. For multiples of five print chicken instead of the number. For the multiples of seven print monkey. For numbers which are multiples of both five and seven print chickenmonkey.

Take a Number - Battle of the Bands

Write a function that accepts any band name as an argument. The function will increment a global variable by one each time it is invoked, and return that number, and the band name concatenated together.

Cookout

Your task is to iterate over the array of foods and invoke the function for each item so that the cookedFood array contains all of the items after they are cooked.

Overly Excited

For this exercise, you need to create a single JavaScript function named addExcitement that will use console.log() to print out a sentence to the browser console.

Stacking Words

The learning objective of this exercise is to use iteration and string concatenation. Create an addExcitement function that should console.log() rows of words. It should take an array containing the words of a sentence and output them in the developer console.

Some words are more exciting than others

Add logic to addExcitement that places an exclamation point (!) after every third word. If the current value of the counter variable can be evenly divided by 3 (using the JavaScript remainder operator) add a single exclamation point (!) to the current word in the array.
